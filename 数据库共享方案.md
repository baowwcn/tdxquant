# 数据库共享方案

本文档列出在局域网环境中，从 Mac (192.168.1.108) 访问放在 Windows (192.168.1.111) 上的 `a_share_full.db` 的可行方案、优缺点和具体操作步骤。

---

## 方案概览（按推荐顺序）

1. Flask API（推荐） — 在 Windows 上运行服务，所有 SQLite 访问由服务负责，Mac 通过 HTTP 查询数据。优点：并发安全、可控、易于添加认证和限流。缺点：需要运行服务并开放端口或使用 SSH 隧道。
2. SMB 文件共享（快速、简单） — 直接把包含 `a_share_full.db` 的文件夹共享给 Mac 并在 Finder 挂载。优点：实现简单，无需额外开发。缺点：SQLite 在网络共享上并发写入有高风险（可能损坏数据库），不推荐并发写场景。
3. SSH 隧道（加密通道） — 在 Windows 启用 OpenSSH，并通过 Mac 建立端口转发到 Windows 上运行的本地服务（如 Flask）。优点：流量加密、无需在防火墙开公网端口。缺点：需配置 SSH 登录并保持隧道连接。
4. 迁移到服务器数据库（推荐长期方案） — 把 SQLite 数据导入到 Postgres/MySQL，在 Windows 上部署数据库服务器并允许局域网访问。优点：并发写读、稳定、可扩展。缺点：迁移工作量和运维成本。

---

## 方案 A — Flask API（详细步骤与示例）

适用场景：多用户查询并发、需要集中控制和安全性时。

1. 在 Windows（192.168.1.111）中工作目录存放 `a_share_full.db`。
2. 在该目录创建 `sqlite_api.py`（示例）：

```python
from flask import Flask, request, jsonify
import sqlite3

app = Flask(__name__)
DB = r"E:\new_tdx64\PYPlugins\user\a_share_full.db"

def query(sql, params=()):
    conn = sqlite3.connect(DB)
    conn.row_factory = sqlite3.Row
    cur = conn.cursor()
    cur.execute(sql, params)
    rows = [dict(r) for r in cur.fetchall()]
    conn.close()
    return rows

@app.route('/query')
def q():
    code = request.args.get('code')
    limit = int(request.args.get('limit', '60'))
    if not code:
        return jsonify({'error':'missing code'}), 400
    rows = query('SELECT date,open,high,low,close,volume,amount FROM stock_data WHERE code=? ORDER BY date DESC LIMIT ?', (code, limit))
    return jsonify(rows)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

3. 安装依赖并运行（在已有虚拟环境中）：

```powershell
# 激活虚拟环境（示例）
E:\new_tdx64\PYPlugins\user\.venv-1\Scripts\Activate.ps1
pip install flask
python sqlite_api.py
```

4. 在 Windows 防火墙允许 5000 端口或使用 SSH 隧道（见方案 C）。

5. 在 Mac 端请求：

```bash
curl "http://192.168.1.111:5000/query?code=600726.SH&limit=60"
```

安全与部署建议：使用 `gunicorn`/`waitress` 作为生产 WSGI 守护进程，添加基于令牌的认证或 IP 白名单，或将服务放在反向代理 + TLS 之后。

## 方案 B — SMB 文件共享（步骤）

适用场景：只读或偶尔写入且只有单一客户端访问时快速共享。

1. 在 Windows 以管理员身份运行 PowerShell 创建共享：

```powershell
New-SmbShare -Name AShareDB -Path "E:\new_tdx64\PYPlugins\user" -FullAccess "Everyone"
Enable-NetFirewallRule -DisplayGroup "File and Printer Sharing"
```

2. 在 Mac Finder -> Go -> Connect to Server，输入：

```
smb://192.168.1.111/AShareDB
```

登录后，路径将挂载为 `/Volumes/AShareDB/a_share_full.db`，可在 Mac 上直接用 Python 打开。示例 Python 打开路径：

```python
import sqlite3
conn = sqlite3.connect('/Volumes/AShareDB/a_share_full.db')
```

注意：避免多台机器并行写入；若必须远程写操作，改用方案 A 或迁移数据库。

## 方案 C — SSH 隧道（用于加密通道）

1. 在 Windows 上启用 OpenSSH Server（Settings -> Optional Features -> OpenSSH Server），并允许对应端口。
2. 在 Windows 上按方案 A 启动本地服务（监听 `localhost:5000` 或 `0.0.0.0:5000`）。
3. 在 Mac 上建立隧道：

```bash
ssh -L 5000:localhost:5000 your_windows_user@192.168.1.111
# 然后在 Mac 上访问 http://localhost:5000/query?code=600726.SH&limit=60
```

优点：无需在防火墙开放对外端口，所有流量经 SSH 加密。

## 方案 D — 迁移到 Postgres（长期方案）

适用场景：需要稳定并发写入、HTTP/API + SQL 复杂查询或多人同时使用时。

步骤要点：

- 在 Windows 或局域网内的 Linux 服务器上安装 Postgres。
- 使用 Python 把 SQLite 导入 Postgres（示例库 `pgloader` 或自写脚本使用 `pandas`/`sqlalchemy`）。
- 配置 Postgres 的认证与网络访问，Mac 直接用 `psycopg2` 或 `psql` 连接查询。

迁移示例（简要）：

```python
# 使用 pandas + sqlalchemy（仅示例）
from sqlalchemy import create_engine
import pandas as pd
s_conn = 'sqlite:///E:/new_tdx64/PYPlugins/user/a_share_full.db'
d_conn = 'postgresql://user:pass@localhost:5432/a_share'
se = create_engine(s_conn)
de = create_engine(d_conn)
for chunk in pd.read_sql_table('stock_data', se, chunksize=100000):
    chunk.to_sql('stock_data', de, if_exists='append', index=False)
```

## 风险与注意事项

- 绝不要让多台主机同时直接在网络共享路径上并发写 `a_share_full.db`，会造成 sqlite 文件损坏。
- 若仅做查询（只读），SMB 对小规模单客户端使用通常可行，但仍需小心锁竞争。
- 对外网络或非受信网络请加认证与加密（HTTPS 或 SSH 隧道）。

---

## 推荐

- 若你只是想让 Mac 安全稳定地访问数据，我建议先使用 **方案 B（快速测试）** 挂载共享验证路径可读性，然后尽快切换到 **方案 A（Flask API）** 或 **方案 D（Postgres）** 作为长期方案。
